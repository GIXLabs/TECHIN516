# Issue Template

### Issue Title: [Lab Number] - Brief Description

**Lab Number:** (e.g., Lab 0, Lab 1, etc.)

**Description:**
Please provide a clear and concise description of the issue you're experiencing. Include specific details about the problem, what you were trying to achieve, and any relevant context that might help in resolving the issue.

**Error Messages / Logs:**
If applicable, include the exact error messages or logs you encountered. This information is crucial for diagnosing issues related to code or system errors.

**Screenshots / Images:**
Attach any screenshots or images that help illustrate the issue. This could include:
- Screenshots of the error messages
- Images of robot configurations or setups
- Visual output from simulations or code

**Steps to Reproduce:**
Detail the steps someone would need to take to replicate the issue. This should include:
- Specific code snippets or commands run
- Environmental setups or configurations used
- Any other relevant procedures

**Expected Behavior:**
Describe what you expected to happen when you performed the steps above.

**Actual Behavior:**
Describe what actually happened. Include the full output of any commands run, if applicable.

**Attempts to Resolve:**
List any steps you have already taken in an attempt to resolve the issue. This helps avoid duplication of effort and may provide insight into the problem.
